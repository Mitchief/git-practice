# Day 02 - Git & GitHub

## The core workflow (do this every time)
1. Make changes
2. git add . (stage everything)
3. git commit -m "message" (save snapshot)
4. git push (send to GitHub)

## Commands I learned
- git init: start tracking a folder
- git status: see what changed (red = unstaged, green = staged)
- git diff: see exactly what lines changed
- git add . : stage all changes
- git commit -m "message": save a snapshot
- git log --oneline: see commit history
- git push: send to GitHub
- git remote add origin [url]: connect local to GitHub (one time)
- git checkout -b [name]: create and switch to new branch
- git checkout [name]: switch between branches
- git merge [name]: bring branch changes into current branch
- git branch: list all branches
- git branch -d [name]: delete a branch

## Key concepts
- > replaces file contents, >> adds to them
- git add . stages everything (the dot means all)
- Branches let you experiment without risking main code
- First push needs -u origin main, after that just git push
